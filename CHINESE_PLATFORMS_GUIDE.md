## 🔧 技术实现细节

### 哔哩哔哩解析原理

```javascript
// 1. 获取 window.__playinfo__ 数据
// 2. 解析 DASH 格式（分离的视频和音频）
// 3. 或解析 FLV 格式（合并的视频）
// 4. 提取直接下载链接
```

**支持的格式：**
- 视频流：H.264, H.265 (HEVC)
- 音频流：AAC, FLAC
- 容器：MP4, FLV

### 抖音解析原理

```javascript
// 1. 获取 RENDER_DATA 元素
// 2. 解码 URL 编码的数据
// 3. 递归查找视频数据对象
// 4. 提取 playAddr 或 playApi
```

**支持的格式：**
- 视频编码：H.264
- 容器：MP4

### 小红书解析原理

```javascript
// 1. 获取 window.__INITIAL_STATE__
// 2. 定位 note 或 noteDetailMap
// 3. 提取 video 或 videoMedia
// 4. 解析 streaming 对象中的 H.264/H.265 流
```

**支持的格式：**
- 视频编码：H.264, H.265
- 容器：MP4

## ⚠️ 常见问题

### Q: 为什么有些视频下载失败？

**可能原因：**
1. **需要登录** - 部分内容需要账号登录
2. **防盗链** - 视频有防盗链保护
3. **地理位置限制** - 某些内容仅限特定地区
4. **页面结构变化** - 平台更新了页面结构

**解决方案：**
- 尝试其他视频
- 检查网络连接
- 等待 App 更新支持新的页面结构

### Q: 下载的视频有水印怎么办？

**抖音：**
- 部分视频源提供无水印版本
- 如果下载的有水印，说明源视频就有水印

**小红书：**
- 通常无水印
- 如果有水印，可能是平台添加的

### Q: 下载速度很慢怎么办？

**优化建议：**
1. 使用 WiFi 连接
2. 避免高峰时段下载
3. 选择较低画质（如果可选）
4. 检查网络带宽

### Q: 支持下载直播回放吗？

**支持情况：**
- ✅ 哔哩哔哩：支持（如果已结束并生成回放）
- ⚠️ 抖音：部分支持（取决于是否生成回放）
- ⚠️ 小红书：不支持（小红书没有直播功能）

### Q: 支持下载合集/播放列表吗？

**当前状态：**
- ❌ 暂不支持批量下载
- ✅ 可以逐个下载
- 🔜 未来版本可能支持

## 🎯 最佳实践

### 哔哩哔哩
1. **选择合适画质**
   - 1080P 适合大多数场景
   - 4K 需要足够存储空间
   
2. **注意文件大小**
   - 高画质视频可能很大
   - 检查 iPhone 存储空间

3. **使用 BV 号**
   - BV 号链接更稳定
   - 避免使用临时分享链接

### 抖音
1. **使用完整链接**
   - 包含视频 ID 的完整链接
   - 避免使用短链接（可能过期）

2. **及时下载**
   - 抖音视频可能会下架
   - 及时保存喜欢的视频

3. **检查视频完整性**
   - 下载后预览检查
   - 确保无损坏

### 小红书
1. **区分笔记类型**
   - ✅ 视频笔记：可以下载
   - ❌ 图文笔记：无法下载
   - ⚠️ 视频+图文：仅下载视频部分

2. **注意版权**
   - 尊重创作者版权
   - 仅供个人使用

3. **使用官方链接**
   - 避免使用第三方生成的链接
   - 使用小红书 App 内的分享链接
